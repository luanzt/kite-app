# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a polished dark mode across every Kite screen, driven by per-theme color tokens, with a Light / Dark / System selector in Settings.

**Architecture:** Three layers landed in order — (1) restructure the color tokens in `global.css` into per-theme `@variant light/dark` blocks so all class-driven color flips automatically; (2) a `useThemeColors()` hook + theme-aware pace palette to fix the ~116 hex / ~60 `color=` sites that bypass classes (icons, then pace palette, then charts); (3) widen the store to `light|dark|system`, resolve `system` via `useColorScheme()`, and swap the Settings toggle for a 3-way `Segmented`.

**Tech Stack:** React Native CLI, Uniwind (Tailwind v4 for RN), HeroUI Native, Zustand + MMKV, react-native-svg, Jest.

## Global Constraints

- Styling: use Tailwind `className`, NEVER inline `style={{…}}`; NEVER interpolate a value into a class string (renders unstyled). Branch the whole class on bool/enum. `style` only for runtime-continuous values or host components that reject `className`; when used, via `StyleSheet.create()`.
- Text: use `<Typography>` from `heroui-native`, never `<Text>`.
- Icons: only `lucide-react-native`, sized/colored via `size`/`color` props.
- Overlays: HeroUI Native (`BottomSheet`/`Dialog`/`Popover`), never RN `Modal`.
- i18n: keep `src/i18n/locales/en.json` and `vi.json` key-for-key in sync. Never hardcode visible strings.
- TypeScript strict mode; `yarn tsc` must be clean. `yarn lint` must pass.
- op-sqlite is mocked in Jest; DB-calling code is NOT unit-tested (verify on simulator).
- All Bash commands must run from the repo root `/Users/anntt/Documents/Kite` (a prior skill left the shell cwd elsewhere — always `cd` to root or use `git -C`).

```typescript
export type ResolvedTheme = 'light' | 'dark'
export type ThemeMode = 'light' | 'dark' | 'system'

/**
 * Resolve the user's theme preference to a concrete theme. `system` follows the
 * OS color scheme (from RN useColorScheme, which may be null), defaulting to
 * light when the scheme is unknown.
 */
export function resolveTheme(
  mode: ThemeMode,
  systemScheme: 'light' | 'dark' | null
): ResolvedTheme {
  if (mode === 'system') return systemScheme === 'dark' ? 'dark' : 'light'
  return mode
}
```

- [ ] **Step 1: Rewrite the hook.**

Replace `src/hooks/useTheme.ts` with:

```typescript
import { useEffect } from 'react'
import { useColorScheme } from 'react-native'
import { Uniwind } from 'uniwind'
import { useAppStore } from '@store/useAppStore'
import { resolveTheme } from '@hooks/resolveTheme'

/**
 * Bridges the persisted themeMode preference to Uniwind's active theme.
 * `system` follows the OS color scheme live. Returns the resolved concrete
 * theme so components (and useThemeColors) can pick theme-aware hex values.
 */
export function useTheme() {
  const themeMode = useAppStore((s) => s.themeMode)
  const setThemeMode = useAppStore((s) => s.setThemeMode)
  const systemScheme = useColorScheme() // 'light' | 'dark' | null
  const resolvedTheme = resolveTheme(themeMode, systemScheme ?? null)

  useEffect(() => {
    Uniwind.setTheme(resolvedTheme)
  }, [resolvedTheme])

  return {
    themeMode,
    resolvedTheme,
    isDark: resolvedTheme === 'dark',
    setThemeMode
  }
}
```

```typescript
import type { PaceStatus } from '@features/trackers/types'
import { paceColorsFor } from '@features/trackers/icons'
import { useTheme } from '@hooks/useTheme'

/**
 * Concrete hex values for the active theme, for use where a Tailwind class
 * can't reach — react-native-svg fill/stroke and lucide `color=` props. Keeps
 * these sites in sync with the token palette in global.css by mirroring the
 * same light/dark values here.
 */
const LIGHT = {
  ink: '#1b1e18',
  ink2: '#565a4f',
  ink3: '#8a8e80',
  brand: '#2456b5',
  brandProjected: '#5b8af0',
  line: '#e3e5dc',
  lineStrong: '#d2d5c8',
  gridFaint: '#eef1f5',
  surface: '#ffffff',
  onAccent: '#ffffff'
}

const DARK: typeof LIGHT = {
  ink: '#e8eaed',
  ink2: '#a8adb5',
  ink3: '#6f757e',
  brand: '#5b8def',
  brandProjected: '#89b0f4',
  line: '#2c2f36',
  lineStrong: '#3a3e47',
  gridFaint: '#24272d',
  surface: '#1c1f24',
  onAccent: '#ffffff'
}

export function useThemeColors() {
  const { resolvedTheme, isDark } = useTheme()
  const base = isDark ? DARK : LIGHT
  const pace = paceColorsFor(resolvedTheme)
  return {
    isDark,
    ...base,
    pace: pace.color as Record<PaceStatus, string>,
    paceWeak: pace.weak as Record<PaceStatus, string>
  }
}
```
